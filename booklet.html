About This Document

This booklet accompanies the live performance of RIPPLES: Operative Ecologies in the Worldtext, presented at ELO 2026 in Atlanta, GA. It documents the system's architecture, its core concepts, and the patterns used to build its interfaces.

The project explores imaginary relationalities‚Äîthe ways entities in nonhuman ecologies might perceive, interact with, and be affected by their environment. Rather than simulating "realistic" ecologies, RIPPLES creates a space for speculation and hallucination.

1. Introduction: The Worldtext

The Worldtext is the core of RIPPLES. It is a text-based representation of an imaginary ecology, where each entity is a word or phrase, and each interaction is a change in the text.

WORLDTEXT_GENERATOR: ACTIVE
ENTITY: FORMICIDAE_SCOUT (PERSPECTIVE_LOCKED)
VECTOR: GOAL (SUSTENANCE/SUGAR_TRACE)
RIPPLE_EFFECT: INITIATED

The Worldtext is not a narrative. It is a state readout‚Äîa snapshot of the ecology at a given moment. The user does not write the story; the user injects vectors that shift the state.

Perspective: Formicidae Scout

The stillness is broken by micro-vibrations. The Ant entity abandons the boundary cracks, navigating the ceramic topography of the 'twenty plates'. It traces an invisible chemical scent-line toward the 'six short glasses' where a residue of dried liquid remains. The 'dust' text blocks are disturbed in its wake.

2. Core Concepts

Entities

An Entity is any object, organism, or force in the ecology. Entities have properties like mass, position, and state. They can be selected as the perspective for a ripple.

Vectors

A Vector is a force or direction applied to an entity. There are three core vectors:

SHIFT
Change direction or state. A perturbation.
GOAL
Move toward a resource or target.
OBSTACLE
Encounter a barrier or threat.

The Ripple

When a vector is injected into an entity, it creates a Ripple‚Äîa cascade of state changes that propagate through the ecology. The Ripple updates the Worldtext.

4. Supervised Mode (Human-in-the-Loop)

In Supervised Mode, the human operator manually selects an entity and injects a vector. The system waits for input before triggering a ripple.

Step 1: Select Entity
Step 2: Choose Vector
Step 3: Observe Ripple

Workflow

  1. SELECT ENTITY ‚Äî Click an entity in the Entity Pool to set the perspective.
  2. INJECT VECTOR ‚Äî Click SHIFT, GOAL, or OBSTACLE to apply a force.
  3. TRIGGER RIPPLE ‚Äî The Worldtext updates. Observe the result in the Audit Log.

8. System Architecture

System Overview Diagram
Entity Pool & Control Panel

Key Components

  • SCENARIO_SELECT ‚Äî Choose from preset ecologies or create a custom world.
  • ENTITY_POOL ‚Äî A clickable list of entities. Sets selectedEntity state.
  • VECTOR_CONTROLS ‚Äî Three buttons: SHIFT (purple), GOAL (gold), OBSTACLE (red).
  • WORLDTEXT_READOUT ‚Äî A syntax-highlighted text area displaying the current state.
  • AUDIT_LOG ‚Äî A scrolling list of timestamped events.
  • AUTOPLAY_TOGGLE ‚Äî Enables/disables autonomous mode.
